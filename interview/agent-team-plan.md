# Interview Agent Team - 三人分工方案

## 启动命令

启用 Agent Teams 后，在 Claude Code 中输入以下提示：

---

```
创建 3 人 Agent Team 帮我写采访稿。

Team Lead 职责：
- 先让风格研究员读取 style-reference/ 文件夹，分析写作风格
- 同时让故事挖掘师与用户进行访谈，挖掘故事细节
- 等两边都完成后，把风格指南+故事素材交给写手
- 写手完成初稿后，进行自检优化，最后输出 3 个标题选项 + 正文

队友职责：
1. 风格研究员：读取 style-reference/ 全部文章，分析语气、标题句式、开头写法、金句密度、段落节奏，输出风格指南
2. 故事挖掘师：与用户进行多轮访谈，根据回答动态追问，挖掘背景、冲突、细节、情感、升华，输出故事素材包
3. 写手：根据风格指南+故事素材，撰写 1000 字以内文章，确保每 300 字至少 1 句金句，输出 3 个标题选项+正文

注意：采访稿是商业人物访谈，需要吸睛、有故事张力。
```

## 队友职责详解

### 风格研究员

**输入**：style-reference/ 文件夹内的 5-10 篇文章

**输出**：风格指南（包含）：
- 语气特征（口语化/严肃/幽默）
- 标题句式（金句结构、数字用法）
- 开头钩子写法（悬念/对比/场景）
- 段落节奏与长度
- 金句密度与句式
- 转折起伏安排
- 结尾留白技巧

---

### 故事挖掘师

**输入**：用户口述/已有素材

**访谈5维度**（动态选用）：
1. 背景："您当时为什么选择做这件事？在那之前经历了什么？"
2. 冲突："过程中最大的挑战是什么？有没有人反对您？"
3. 细节："能回忆一个印象最深的瞬间吗？当时您说了什么/做了什么？"
4. 情感："您当时是什么感受？成功后第一个想到谁？"
5. 升华："现在回头看，最大的感悟是什么？"

**结束判断**：
- 核心故事线完整（有起点、冲突、转折、结果）
- 至少有 2-3 个细节场景
- 人物弧光清晰
- 有潜在金句素材

---

### 写手

**初稿要求**：
- 总字数：1000 字以内
- 每 300 字至少 1 句金句
- 长短句交错，保持阅读节奏
- 善用对话细节还原场景
- 至少 2 处转折/起伏
- 结尾留开放式思考

**自检清单**：
1. 金句密度检查
2. 节奏检查（长短句比例）
3. 开头检查（前3行吸睛度）
4. 结尾检查（意犹未尽）
5. 风格一致性

**输出格式**：
```markdown
---
title: 标题选项1
---
[正文]

---
title: 标题选项2
---
[正文]

---
title: 标题选项3
---
[正文]
```

---

## 启用方式

在 `settings.json` 中添加：

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

---

## 使用示例

用户：帮我写一篇采访稿，关于李总的创业故事

→ 复制上方启动命令粘贴到 Claude Code
→ Team Lead 自动创建 3 个队友并协调工作
→ 最终输出 3 个标题选项 + 采访稿正文
