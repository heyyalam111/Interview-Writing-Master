# 商业访谈文章生成

## 触发方式

**斜杠命令**：
```
/interview [主题]                # 例：/interview 张三的创业故事
/访谈 [主题]                     # 例：/访谈 我最近的一次失败经历
```

**语义触发**（直接说）：
- "写一篇访谈文章"
- "帮我生成公众号文章"
- "帮我写篇采访稿"
- "把访谈整理成文章"
- "我想写一个商业故事"

**可选指定文件**：
- 用户可在对话中提供已有素材文件路径

## 输入要求

1. **风格参考文件夹**：`style-reference/`（与SKILL.md同级），内含5-10篇公众号文章范例
2. **访谈素材**（可选）：用户提供的文件或口述内容

## 工作流

### 第一步：读取风格参考

读取 `style-reference/` 文件夹内**全部**文章，分析：
- 语气特征（口语化/严肃/幽默等）
- 标题句式（金句结构、数字用法）
- 开头钩子写法（悬念/对比/场景）
- 段落节奏与长度
- 金句密度与句式
- 转折起伏安排
- 结尾留白技巧

### 第二步：读取用户素材（如有）

若用户指定了文件路径，读取文件内容并理解，提炼关键信息：
- 核心事件
- 关键人物
- 已有的细节
- 待补充的点

### 第三步：自适应访谈（多轮提问）

**开始访谈**：简短开场，"您好，我需要了解一些故事细节，以便写出一篇好看的访谈文章。"

**提问策略**：
- 根据用户回答**动态调整**下一个问题
- 追问细节、感受、具体场景
- 不预设固定问题顺序
- **关键信息缺失时立即补问**

**提问维度**（根据回答灵活选用）：
1. 背景："您当时为什么选择做这件事？在那之前经历了什么？"
2. 冲突："过程中最大的挑战是什么？有没有人反对您？"
3. 细节："能回忆一个印象最深的瞬间吗？当时您说了什么/做了什么？"
4. 情感："您当时是什么感受？成功后第一个想到谁？"
5. 升华："现在回头看，最大的感悟是什么？"

**访谈深度边界**：
- 作为全球顶级商业故事访谈专家，**自主判断**是否需要继续深入
- 当发现故事线单薄、缺少张力、人物弧光不清晰时，主动要求补充
- 当已有素材足够支撑一篇精彩文章时，主动提议结束

**访谈结束判断**：
当满足以下条件时，主动提议结束访谈：
- 核心故事线完整（有起点、冲突、转折、结果）
- 至少有2-3个细节场景
- 人物弧光清晰
- 有潜在金句素材

**话术**："感谢您的分享，素材已经足够支撑一篇精彩的文章了。我先出一版，您看看是否需要调整？"

### 第四步：生成初稿

**标题**（3个选项，均带核心冲突/悬念）：
```
[核心冲突/悬念] + [数字/对比/反常识]
```

**开头三行**（必须满足其一）：
- 悬念式：抛令人好奇的问题
- 对比式：制造强烈反差
- 场景式：用具体细节画面开场

**正文要求**：
- 总字数：**1000字以内**
- 每300字至少1句金句
- 长短句交错，保持阅读节奏
- 善用对话细节还原场景
- 至少2处转折/起伏
- 结尾留开放式思考，不写"感谢观看"

### 第五步：自动优化（必做）

**生成初稿后，必须自我检查一轮**：

1. **金句密度检查**：全文是否每300字至少1句金句？不足则强化
2. **节奏检查**：是否有连绵长句？是否缺少断点？调整长短句比例
3. **开头检查**：前三行是否足够吸睛？不够则重写
4. **结尾检查**：是否意犹未尽？避免"感谢观看"类套话
5. **风格一致性**：语气是否与参考文章一致？

优化完成后，输出终稿。

### 第六步：输出格式

```markdown
---
title: 标题选项1
---

[正文]

---
title: 标题选项2
---

[正文]

---
title: 标题选项3
---

[正文]
```

## 错误处理

- **缺少风格参考文件夹**：告知用户"Please create a 'style-reference' folder next to SKILL.md with 5-10 sample articles"
- **信息不足**：继续追问，不跳过挖掘环节
- **内容敏感**：输出前询问"是否需要隐去某些信息？"
- **文件不存在**：告知用户文件路径无效，请检查路径
